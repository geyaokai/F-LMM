# 第四章 解释单元的概念化与知识对齐

## 4.1 从区域证据到概念解释

单纯给出 bbox 或 mask 虽然已经比 heatmap 更具可验证性，但这仍然主要是像素层解释。为了进一步提升人类可理解性，本文引入“解释单元”概念，将 grounding 得到的区域进一步映射到可读的视觉概念和回答片段，从而形成更完整的解释结构。

本文所说的解释单元，统一表示为：

`图像区域 - 视觉概念 - 回答片段 - 证据元数据`

## 4.2 mention 到 concept 的归一化

回答中的短语往往包含颜色、方位、数量或修饰信息，例如 “yellow taxi” 或 “man's shirt”。对于解释任务而言，更关键的是其中的核心视觉概念，如 “taxi” 或 “shirt”。因此，本节需要介绍如何从回答中的 mention 抽取和归一化出 concept，并保留 `mention_text / concept_text / concept_span / concept_source` 等字段。

这一部分可以结合你当前的 phrase candidate 与 concept normalizer 实现来写。

## 4.3 基于 VCD 的概念对齐

在 concept 被抽取出来之后，可以先与 VCD 中已有视觉概念或属性进行对齐。VCD 的意义在于为解释提供一个更稳定、更具视觉先验的概念空间，使 grounding 结果不只是“框住了某一块区域”，而是能进一步说明“这块区域被理解成什么概念”。

## 4.4 基于 WordNet 的视觉概念补充与差集分析

仅依赖 VCD 可能存在概念覆盖不足的问题，因此本文进一步考虑引入 WordNet 作为概念扩展资源。本节可以写成一个相对独立的分析部分，重点回答两个问题：

1. WordNet 中能够提取出多少与视觉实体相关的概念。
2. 与 VCD 相比，WordNet 额外提供了哪些概念，它们在数量和类别构成上有何特点。

建议本节输出至少三类结果：

- WordNet 视觉概念总量
- `WordNet - VCD` 差集规模
- 差集概念的大体构成，例如实体类、场景类、器具类、属性相关类等

这样一来，WordNet 就不是简单的“再加一个词库”，而是服务于概念层解释空间扩展的分析组件。

## 4.5 解释单元的统一表示

在完成 concept 抽取和知识对齐之后，可以将每条解释记录统一表示为：

- 区域位置或掩码
- mention_text
- concept_text
- answer span / token span
- concept 来源
- 是否有 grounding 证据
- 是否触发纠错

这一统一格式能够直接服务于后续诊断、人工复核和前端展示。

## 4.6 本章小结

本章从 grounding 区域出发，引入了解释单元的概念化思路，并讨论了基于 VCD 和 WordNet 的知识对齐方法，使解释从像素层上升到了概念层，为后续诊断与纠错提供了更高层次的语义基础。
