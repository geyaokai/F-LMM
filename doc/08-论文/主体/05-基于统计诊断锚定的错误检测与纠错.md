# 第五章 基于统计诊断锚定的错误检测与纠错

## 5.1 问题动机

对于多模态大模型而言，错误回答并不总是因为完全“看错了图像”，还可能来自语言先验过强、历史输出自回归惯性过强、或回答虽然流畅但缺少视觉证据支撑。若解释系统只能事后展示区域，而不能进一步判断这些区域是否真正支撑了回答，那么可解释性仍然停留在展示层。

因此，本文进一步考虑：能否把 grounding、concept 和 region-to-token 等解释信号统一起来，作为诊断和纠错的依据。

## 5.2 统计意义上的概念诊断锚定方案

本节建议作为本章核心方法。所谓“统计意义上的概念诊断锚定”，是指不依赖单一信号做武断判断，而是综合多种结构化证据，对回答是否真正被图像支撑进行诊断。

可以考虑纳入以下诊断因子：

1. grounding 是否存在。
2. grounding 区域是否质量异常，例如为空、偏移或明显不合理。
3. answer mention 与 concept 是否一致。
4. 给定区域后，region-to-token 是否能回看到对应 answer span。
5. 回答是否更像依赖历史文本或语言惯性，而非视觉证据。

在实现上，可以先从规则化、计数式或阈值式方案做起，再逐渐发展为更系统的统计评分方式。

## 5.3 诊断标签设计

为了让诊断结果可复用、可评估、可人工复核，建议将诊断输出规范成少量明确标签，例如：

- `keep`：证据充分，保留当前回答
- `reject`：缺乏关键视觉证据，应拒绝或判定为错误
- `repair`：存在可定位问题，可进入纠错流程

这部分可以与你现有的 correction self-check 结构保持一致。

## 5.4 纠错策略

在被诊断为 `repair` 或 `reject` 后，可进一步触发纠错机制。当前论文中可以先写三类候选策略：

1. ROI re-answer：基于关键区域重新回答。
2. 重采样：当回答不稳定时重新生成并比较证据一致性。
3. 提示重构：显式要求模型基于图像证据重新作答。

中期阶段可以重点写 ROI re-answer，后两类作为后续实验扩展。

## 5.5 自检与人工覆盖机制

为了让纠错链路具备工程可复现性，可以设计 baseline、auto-correction、human-override 三套统一结果格式，并统计 detection failure、judgment failure、repair failure 等失败类型。这样做的价值在于：

- 让每次改动都能定位到是检测层、判定层还是修复层出了问题
- 支持中期阶段先形成稳定评测闭环
- 为后续接入前端交互式人工复核提供统一接口

## 5.6 本章小结

本章围绕“解释如何反过来帮助模型性能和可靠性”展开，提出了基于统计诊断锚定的错误检测与纠错思路，使 visual grounding 不再只是事后说明工具，而是成为模型校准与修正的辅助机制。
